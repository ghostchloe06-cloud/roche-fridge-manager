# 我的冰箱 Roche 插件

一个纯前端 Roche 插件，用原生 JavaScript 写成，不需要 npm、构建工具、React 或 Vue。

## 文件结构

```txt
roche-fridge-manager/
  manifest.json
  fridge_manager_plugin.js
  README.md
```

## 安装链接

上传到 GitHub 后，推荐在 Roche 插件安装处填写 manifest 的 Raw 链接：

```txt
https://raw.githubusercontent.com/ghostchloe06-cloud/roche-fridge-manager/main/manifest.json
```

如果仓库名或用户名不同，把 `manifest.json` 里的 `entry` 也一起改成真实的 Raw JS 链接。

## 功能

- Tab 1「库存」：新增、编辑、删除、筛选、标记冷藏/冷冻/常温、记录数量/单位/分类/到期日。
- Tab 2「本周三餐」：7 天 x 早午晚。可以先手填，留空的格子由 AI 生成。
- 单格重 roll：不喜欢某一餐，可以点该格的「重 roll」单独换掉。
- 购物清单：根据菜单所需食材和当前库存做粗略差异，生成可编辑清单。
- 角色管理：选择一个 Roche char 做三餐管理员，菜单生成时会参考 TA 的人设、简介、说话方式和对应会话长期记忆。
- 偏好记忆：可选额外挂载 Roche 会话长期记忆，菜单生成时会把忌口、偏好、习惯当作参考。
- 健康档案：身高、体重、目标热量、饮食目标只作为软参考，不做精确营养计算。
- 数据备份：支持导出/导入 JSON，防止误删或换设备时丢数据。
- 今日记忆写入：点击「记住今天」后，可把当天三餐写入所选 char 对应会话的 Roche 主 facts 记忆。

## 存储说明

库存、菜单、购物清单、健康档案和挂载会话 ID 都保存在插件自己的 `roche.storage`。卸载插件时，这些私有数据通常会随插件一起清理。

插件不会直接读写 Roche 主 IndexedDB。角色和记忆读取只用于生成菜单时参考人设、饮食偏好和相处习惯。

只有你主动点击「记住今天」并确认后，插件才会调用 `roche.memory.write()` 写入 Roche 主 facts 记忆。该内容不会随插件卸载自动删除。

## 让角色管理三餐

1. 打开「本周三餐」。
2. 在「角色管理」里点「刷新角色列表」。
3. 选择一个 char 作为三餐管理员。
4. 在管理方式里写你希望 TA 怎么管，例如「温柔但严格，晚餐不要太重，尽量高蛋白」。
5. 如需更多偏好来源，可以勾选额外会话记忆。
6. 点「生成空白餐」或某一餐的「重 roll」。
7. 当天菜单确认后，点「记住今天」，让这个 char 记住你今天吃了什么。

生成时插件会读取：

- 当前库存和到期信息。
- 健康档案软参考。
- 被选中 char 的人设/简介/会话长期记忆。
- 额外勾选会话的长期记忆。

## UI 风格

界面按 Roche / NOIR 的黑白移动端风格处理：白底、黑色主按钮、圆形返回/操作按钮、pill tabs、细分割线和灰色辅助信息。

## 风险提示

Roche 插件是全信任 JS。请只安装你信任的插件链接。这个插件会请求读取角色、读取/写入会话记忆和调用 AI 的权限，用途是让指定角色参考你的忌口、习惯和库存来管理三餐，并在你确认后记住当天三餐。
